## 详细分析

### 1. 设备软件包上传

- **风险等级：** 严重
- **来源文档：** 需求规格说明书、运维操作指南
- **章节路径：** 第三章 业务配置 > 3.2 上传设备软件包
- **功能描述：** 管理员通过 Web 界面将设备软件包（固件、补丁文件 .bin/.pat/.zip）上传到网管服务器，用于设备升级。单个文件最大 500MB，上传后系统自动校验完整性。
- **章节片段：** "管理员可通过Web界面将设备软件包（如固件、补丁文件）上传到网管服务器。上传后的软件包可用于设备升级。支持上传的文件格式包括.bin、.pat、.zip，单个文件最大500MB。上传过程中显示进度条，上传完成后系统自动校验文件完整性。"

#### 为什么是高危业务

该功能涉及文件上传操作，属于典型的高危文件/路径操作模式。上传的文件会直接用于设备升级，一旦被恶意利用可导致全网设备被植入恶意固件。上传接口是系统的外部暴露面，且文件类型涉及可执行内容（.bin/.pat 等固件格式）。

#### 业务影响分析

| 攻击场景 | 可能性 | 影响程度 | 业务后果 |
|---------|--------|---------|---------|
| 上传恶意固件包 | 中 | 严重 | 设备被植入后门，全网设备可被攻击者控制 |
| 上传超限文件耗尽存储 | 高 | 中 | 服务器磁盘空间耗尽，导致系统服务异常 |
| 上传过程中文件被篡改 | 低 | 严重 | 设备升级到被篡改的固件导致功能异常 |
| 文件名路径遍历攻击 | 中 | 高 | 文件被写入非预期目录，覆盖系统文件 |

#### 安全测试建议

**测试点 1：文件类型校验**
- **测试类型：** 功能安全验证
- **前置条件：** 已登录系统
- **测试步骤：** 1) 尝试上传非允许格式（.exe/.sh/.jsp）；2) 修改合法文件扩展名上传；3) 上传空文件、0 字节文件
- **预期结果：** 非允许格式被拒绝，系统给出明确提示
- **检查点：** 校验基于文件内容魔术字节而非仅扩展名

**测试点 2：文件上传大小限制**
- **测试类型：** 异常场景测试
- **前置条件：** 已登录系统
- **测试步骤：** 1) 尝试上传超过 500MB 的文件；2) 尝试并发上传多个大文件；3) 上传过程中断开连接
- **预期结果：** 超过限制时系统拒绝并清理临时文件，并发上传有限制
- **检查点：** 上传中断后临时文件是否被清理

**测试点 3：路径遍历防护**
- **测试类型：** 渗透测试
- **前置条件：** 已登录系统
- **测试步骤：** 1) 抓包修改上传文件名包含 `../`；2) 尝试上传到 Web 目录
- **预期结果：** 文件名被过滤或规范化，不存在路径穿越
- **检查点：** 文件存储路径不可被客户端控制

### 2. SFTP备份服务器配置

- **风险等级：** 严重
- **来源文档：** 需求规格说明书、运维操作指南
- **章节路径：** 第二章 安装部署 > 2.3 配置SFTP备份服务器
- **功能描述：** 管理员配置备份文件的远程存储位置，通过 SFTP 协议将设备配置文件备份到远程服务器。配置项包括：SFTP 服务器 IP 地址、端口号、用户名、登录密码、存储目录路径。
- **章节片段：** "管理员可以配置设备配置备份文件的存储位置。支持将备份文件存储在网管服务器本地磁盘，或通过SFTP协议存储到远程备份服务器。配置项包括：SFTP服务器IP地址、端口号、用户名、登录密码、存储目录路径。"

#### 为什么是高危业务

该功能同时涉及**文件/路径操作**（配置备份存储路径、文件传输）和**凭据配置**（SFTP 登录密码），属于复合高危模式。备份文件包含设备的完整配置、SNMP Community 字符串、登录凭据等敏感信息。一旦 SFTP 凭据泄露或备份服务器配置被篡改，后果严重。

#### 业务影响分析

| 攻击场景 | 可能性 | 影响程度 | 业务后果 |
|---------|--------|---------|---------|
| SFTP密码被截获（明文传输/存储） | 中 | 严重 | 备份服务器被未授权访问，全网设备配置泄露 |
| SFTP服务器地址被篡改 | 低 | 严重 | 备份数据发送到攻击者控制的服务器 |
| 备份路径配置为本地敏感目录 | 低 | 高 | 备份文件覆盖系统文件，导致服务异常 |
| 备份数据在传输中被篡改 | 低 | 高 | 恢复恶意配置导致设备故障 |

#### 安全测试建议

**测试点 1：SFTP密码传输和存储安全**
- **测试类型：** 传输安全测试
- **前置条件：** 已进入系统管理后台
- **测试步骤：** 1) 配置 SFTP 密码并保存；2) 抓包检查密码在传输过程中是否加密；3) 检查数据库或配置文件中密码的存储方式（明文/密文/哈希）
- **预期结果：** 密码在传输和存储中均加密，禁止明文存储
- **检查点：** 前端提交时密码已加密、后端存储为密文或哈希

**测试点 2：SFTP连接认证安全**
- **测试类型：** 异常场景测试
- **前置条件：** SFTP 服务器配置完成
- **测试步骤：** 1) 配置错误的密码并测试系统反馈；2) 修改 SFTP 服务器主机密钥测试是否校验；3) 配置不可达的服务器地址测试超时处理
- **预期结果：** 连接失败时有明确提示，不泄露服务器信息
- **检查点：** 错误信息不包含具体认证细节（如"密码错误"vs"用户不存在"）

**测试点 3：路径配置校验**
- **测试类型：** 功能安全验证
- **前置条件：** 已登录系统
- **测试步骤：** 1) 配置路径包含 `../` 尝试越界；2) 配置为系统关键路径（如 `/etc/`）；3) 测试路径不存在时系统的处理
- **预期结果：** 路径被规范化，禁止配置到系统敏感目录
- **检查点：** 路径参数经过白名单校验或规范化处理

### 3. 北向接口对接OSS系统

- **风险等级：** 高
- **来源文档：** 需求规格说明书、第三方系统对接集成指南
- **章节路径：** 第四章 系统对接 > 4.1 北向接口配置
- **功能描述：** 网管平台提供 RESTful 北向接口，供上级 OSS 系统获取全网资源数据。对接配置包括 HTTPS 证书配置、Token 认证参数、数据推送地址设置。
- **章节片段：** "网管平台提供北向接口供上级OSS系统对接。OSS系统可以通过RESTful接口获取全网资源数据。对接配置包括：HTTPS证书配置、Token认证参数、数据推送地址设置。"

#### 为什么是高危业务

该功能涉及**第三方系统对接**（北向接口开放给外部系统调用），属于典型的高危对接模式。接口提供全网资源数据（设备 IP、型号、序列号、配置等敏感信息），是系统最重要的外部数据出口。HTTPS 证书和 Token 认证的配置安全性直接影响接口防护能力。

#### 业务影响分析

| 攻击场景 | 可能性 | 影响程度 | 业务后果 |
|---------|--------|---------|---------|
| Token认证被绕过或伪造 | 中 | 严重 | 未经授权的第三方获取全网数据 |
| HTTPS证书配置不当（自签名/过期） | 高 | 中 | 通信加密不可靠，存在中间人攻击风险 |
| 接口未做访问频率限制 | 高 | 中 | 数据被批量爬取，服务器负载过高 |
| 接口参数注入 | 低 | 高 | 通过查询参数注入获取越权数据 |

#### 安全测试建议

**测试点 1：Token认证安全**
- **测试类型：** 渗透测试
- **前置条件：** 获取有效的 Token
- **测试步骤：** 1) 修改 Token 任意字符测试是否校验；2) 重放旧 Token 测试时效性；3) 尝试 Token 暴力破解；4) 检查 Token 是否绑定来源 IP
- **预期结果：** Token 校验严格、有时效性、支持吊销
- **检查点：** Token 失效后接口返回 401，不泄露任何数据

**测试点 2：HTTPS证书配置验证**
- **测试类型：** 功能安全验证
- **前置条件：** 证书配置完成
- **测试步骤：** 1) 使用过期证书测试是否被拒绝；2) 使用自签名证书测试是否告警；3) 测试证书吊销检测机制
- **预期结果：** 证书异常时有告警，支持证书更换
- **检查点：** 系统支持证书有效性自动检测

**测试点 3：接口访问控制**
- **测试类型：** 权限测试
- **前置条件：** 接口已对接
- **测试步骤：** 1) 测试不同 Token 权限级别的数据隔离；2) 测试接口频繁调用是否有频率限制；3) 测试查询参数注入
- **预期结果：** 严格权限隔离，有频率限制，参数经过校验
- **检查点：** 接口响应中不包含超过授权范围的数据字段

### 4. SNMP参数配置

- **风险等级：** 高
- **来源文档：** 需求规格说明书、运维操作指南
- **章节路径：** 第三章 业务配置 > 3.3 SNMP参数配置
- **功能描述：** 管理员配置设备 SNMP 参数，支持 SNMP v1/v2c/v3 版本，配置内容包括 SNMP Community 字符串（只读/读写）、SNMP v3 用户认证密码和加密密码。
- **章节片段：** "管理员可配置设备SNMP参数，包括SNMP版本（v1/v2c/v3）、SNMP Community字符串（只读/读写）、SNMP v3用户认证密码和加密密码。通过SNMP协议，网管系统可读取设备运行状态和性能数据。"

#### 为什么是高危业务

该功能涉及**凭据配置**（Community 字符串、认证密码、加密密码），属于高危模式。SNMP v1/v2c 的 Community 字符串相当于设备管理密码，以明文传输。读写 Community 可被攻击者用于读取和修改设备配置。实际环境中 v1/v2c 仍在广泛使用，暴露面大。

#### 业务影响分析

| 攻击场景 | 可能性 | 影响程度 | 业务后果 |
|---------|--------|---------|---------|
| Community字符串明文传输被截获 | 高 | 严重 | 攻击者获取设备管理权限（v1/v2c无加密） |
| 读写Community泄露 | 中 | 严重 | 攻击者修改设备配置导致业务中断 |
| Community字符串使用默认值 | 高 | 高 | 典型攻击入口，可批量发现和入侵设备 |
| SNMP v3密码配置弱口令 | 中 | 高 | 认证和加密被破解 |

#### 安全测试建议

**测试点 1：Community字符串安全**
- **测试类型：** 功能安全验证
- **前置条件：** 已登录系统，SNMP配置页面
- **测试步骤：** 1) 配置时勾选"显示密码"检查前端是否掩码；2) 保存后查看页面源码是否回显明文；3) 抓包检查 Community 传输是否加密；4) 检查数据库存储方式
- **预期结果：** 界面掩码显示、传输加密、存储密文
- **检查点：** 禁止明文传输、禁止前端/API 返回明文 Community 字符串

**测试点 2：默认Community检测**
- **测试类型：** 功能安全验证
- **前置条件：** SNMP 配置页面
- **测试步骤：** 1) 触发配置向导检查是否使用默认值（public/private）；2) 测试系统是否强制要求修改默认 Community
- **预期结果：** 系统拒绝或告警默认 Community 字符串
- **检查点：** 首次配置或重置后引导用户设置非默认值

**测试点 3：SNMP v3密码策略**
- **测试类型：** 功能安全验证
- **前置条件：** SNMP v3 配置
- **测试步骤：** 1) 设置弱密码（123456）检查策略；2) 测试认证密码和加密密码是否可以相同；3) 测试密码最小长度限制
- **预期结果：** 密码复杂度策略生效，认证和加密密码独立
- **检查点：** 密码策略符合等保要求（长度、复杂度、有效期）

**测试点 4：协议版本控制**
- **测试类型：** 功能安全验证
- **前置条件：** SNMP 配置页面
- **测试步骤：** 1) 检查系统是否允许同时启用 v1/v2c/v3；2) 检查是否可单独禁用 v1/v2c；3) 如果允许 v1/v2c，检查是否有安全提示
- **预期结果：** 系统默认推荐 v3，启用 v1/v2c 时有安全风险提示
- **检查点：** 支持仅启用 v3，v1/v2c 有明确的安全告警

### 5. 告警推送Webhook

- **风险等级：** 高
- **来源文档：** 第三方系统对接集成指南、运维操作指南
- **章节路径：** 第四章 系统对接 > 4.2 告警推送设置
- **功能描述：** 网管平台将设备告警信息通过 Webhook 推送到第三方系统。管理员配置 Webhook 接收地址、推送内容格式（JSON/XML）、推送频率和过滤规则。
- **章节片段：** "系统支持将设备告警信息通过Webhook方式实时推送到第三方运维平台。管理员可配置Webhook接收URL、推送内容格式（JSON/XML）、推送频率和告警过滤规则。支持断线重连和消息队列机制，确保告警不丢失。"

#### 为什么是高危业务

该功能涉及**第三方系统对接**（向外部系统推送数据），属于高危模式。Webhook 机制需要系统主动向外发起 HTTP 请求，存在 SSRF（服务端请求伪造）风险。告警信息包含设备 IP、故障描述、可能含敏感配置信息。Webhook 地址可被篡改导致告警数据泄露给第三方。

#### 业务影响分析

| 攻击场景 | 可能性 | 影响程度 | 业务后果 |
|---------|--------|---------|---------|
| SSRF攻击（设置内网Webhook地址） | 中 | 高 | 攻击者利用服务器探测内网服务 |
| Webhook地址被篡改为恶意服务器 | 低 | 高 | 告警数据（含设备信息）发送给攻击者 |
| 推送消息无签名验证 | 中 | 中 | 第三方系统无法验证消息真实性 |
| 推送频率过高导致第三方系统负载 | 低 | 中 | 第三方系统被大量告警淹没 |

#### 安全测试建议

**测试点 1：SSRF防护**
- **测试类型：** 渗透测试
- **前置条件：** Webhook 配置页面
- **测试步骤：** 1) 配置内网地址（127.0.0.1/10.0.0.1/192.168.x.x）；2) 配置特殊地址（0.0.0.0/[::1]）；3) 测试是否可探测内网端口
- **预期结果：** 内网地址被拒绝或有限制
- **检查点：** Webhook 地址配置有 IP 白名单或内网阻断机制

**测试点 2：Webhook地址校验**
- **测试类型：** 功能安全验证
- **前置条件：** Webhook 配置页面
- **测试步骤：** 1) 配置不可达地址测试超时处理；2) 配置非 HTTP(S) 协议（file:///ftp://）；3) 配置 URL 包含特殊字符
- **预期结果：** 协议限制为 HTTP(S)，无效地址有明确提示
- **检查点：** Webhook 地址经过 URL 格式校验和协议白名单

**测试点 3：消息完整性**
- **测试类型：** 功能安全验证
- **前置条件：** Webhook 推送配置完成
- **测试步骤：** 1) 检查推送消息是否包含签名（HMAC 等）；2) 检查是否支持 HTTPS 推送；3) 测试推送失败的重试和告警机制
- **预期结果：** 推送消息有签名验证，支持 HTTPS，失败有重试
- **检查点：** 签名机制能保证消息未被篡改

### 6. API Token配置

- **风险等级：** 高
- **来源文档：** 第三方系统对接集成指南
- **章节路径：** 第四章 系统对接 > 4.3 API Token配置
- **功能描述：** 北向接口使用 Token 进行认证鉴权。管理员可配置 Token 有效期（默认 24 小时）、刷新策略和访问权限范围。Token 用于所有北向接口调用的身份验证。
- **章节片段：** "北向接口采用Token认证方式，第三方系统需在HTTP请求头中携带有效的Token才能调用接口。管理员可在系统管理界面创建和管理Token，支持设置Token有效期（默认24小时）和访问权限范围。"

#### 为什么是高危业务

该功能涉及**第三方对接**和**凭据配置**，属于复合高危模式。Token 是北向接口的唯一认证凭据，Token 的安全性直接影响所有接口的保护。如果 Token 生成算法可预测、有效期过长、或缺乏吊销机制，攻击者可伪造或盗用 Token 访问系统全部数据。

#### 业务影响分析

| 攻击场景 | 可能性 | 影响程度 | 业务后果 |
|---------|--------|---------|---------|
| Token生成可预测 | 低 | 严重 | 攻击者可伪造有效 Token 访问全部接口 |
| Token有效期过长且无法吊销 | 中 | 严重 | Token泄露后攻击者有充足时间利用 |
| Token未绑定客户端来源 | 中 | 高 | Token 被窃取后可在任意位置使用 |
| Token权限范围过大 | 中 | 高 | 单一 Token 可访问所有资源，违反最小权限原则 |

#### 安全测试建议

**测试点 1：Token生成安全**
- **测试类型：** 渗透测试
- **前置条件：** 获取一批有效的 Token（10个以上）
- **测试步骤：** 1) 分析 Token 格式和编码方式（Base64/JWT/随机串）；2) 尝试解码 Token 查看内容；3) 连续获取 Token 检查是否可预测
- **预期结果：** Token 使用安全的随机算法生成，不可预测、不可解码获取敏感信息
- **检查点：** 使用 JWT 时应验证签名算法为 RS256/HS256 而非 none

**测试点 2：Token生命周期管理**
- **测试类型：** 功能安全验证
- **前置条件：** 已获取有效 Token
- **测试步骤：** 1) 24小时后测试旧 Token 是否生效；2) 测试 Token 吊销接口是否有效；3) 测试 Token 刷新机制；4) 测试重复登录是否吊销旧 Token
- **预期结果：** Token 超过有效期自动失效，支持主动吊销
- **检查点：** 吊销后的 Token 立即不可用，不存在缓存窗口

**测试点 3：Token权限隔离**
- **测试类型：** 权限测试
- **前置条件：** 获取两个不同权限级别的 Token
- **测试步骤：** 1) 使用低权限 Token 调用高权限接口；2) 修改 Token 中的权限声明；3) 测试 Token 是否绑定来源 IP 或 User-Agent
- **预期结果：** Token 遵循最小权限原则，权限变更需要重新认证
- **检查点：** 服务端验证权限而非依赖 Token 中的权限声明

## 横向安全关注点

### 用户权限体系

系统主要面向管理员角色，文档中未明确区分细粒度的角色权限。建议关注以下方面：
- 是否支持基于角色的访问控制（RBAC），不同管理员是否可分配不同权限
- 高危操作（SFTP配置、软件包上传）是否需要二次授权或双人操作
- 是否存在越权风险：A 管理员能否查看或修改 B 管理员的配置

### 数据传输安全

- 北向接口和 Webhook 支持 HTTPS，但需验证证书有效性校验是否严格
- SFTP 协议本身加密，但需验证密码传输（前端到后端）是否加密
- SNMP v1/v2c 的 Community 字符串明文传输，建议仅启用 v3
- Web 管理界面应全站启用 HTTPS，检查 HTTP 到 HTTPS 的重定向

### 数据存储安全

- SFTP 密码、SNMP Community、Token 等敏感凭据必须在存储时加密（非明文）
- 备份文件包含设备敏感配置，建议支持备份文件加密
- 上传的软件包应做完整性校验（哈希校验、数字签名）
- 敏感数据日志中是否做了脱敏处理（密码、Token 等不应记录明文日志）

### 日志与审计

- 高危操作（配置变更、文件上传、系统对接）应有完整的操作日志
- 日志应包括：操作人、操作时间、操作内容、操作结果、来源 IP
- 日志应防篡改（如使用安全日志服务器、日志签名）
- 建议覆盖以下高危操作的审计：SFTP 配置变更、Token 生成和吊销、软件包上传、SNMP 配置变更、Webhook 地址修改
